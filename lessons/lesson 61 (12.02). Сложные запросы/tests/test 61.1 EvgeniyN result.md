# Вопрос 1. Тема: Сложные запросы
  
**1. Какой оператор используется для объединения двух таблиц в SQL?**
  
Варианты ответов:
1) COMBINE
2) MERGE
3) UNION
4) ✅ JOIN
  
### ИИ анализ
Оператор `JOIN` является основным для объединения строк из двух или более таблиц на основе связанного между ними столбца.  Варианты `COMBINE` и `MERGE` не являются стандартными SQL-операторами для этой цели, а `UNION` используется для сложения наборов строк, а не для их объединения по условию.  Вы правильно выбрали `JOIN`, что показывает понимание базового синтаксиса SQL для работы с несколькими таблицами.  

# Вопрос 2. Тема: Сложные запросы
  
**2. Как объединить таблицы Customers и Orders по полю CustomerID и выбрать все строки из таблицы Customers, даже если нет соответствующих записей в таблице Orders?**
  
Варианты ответов:
1) SELECT * FROM Customers RIGHT JOIN Orders ON Customers.CustomerID = Orders.CustomerID
2) SELECT * FROM Customers FULL JOIN Orders ON Customers.CustomerID = Orders.CustomerID
3) SELECT * FROM Customers INNER JOIN Orders ON Customers.CustomerID = Orders.CustomerID
4) ✅ SELECT * FROM Customers LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID
  
### ИИ анализ
`LEFT JOIN` возвращает все записи из левой таблицы (`Customers`) и соответствующие записи из правой (`Orders`).  Если соответствия нет, результат для правой таблицы будет `NULL`.  `INNER JOIN` вернул бы только клиентов с заказами, а `RIGHT JOIN` — наоборот, все заказы, даже без клиентов.  Ваш выбор корректен и демонстрирует понимание различий между типами соединений.  

# Вопрос 3. Тема: Сложные запросы
  
**3. Что вернёт запрос `SELECT Customers.Name, Orders.OrderID FROM Customers LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;`, если в таблице Customers есть клиенты без заказов?**
  
Варианты ответов:
1) Запрос вернёт всех клиентов (включая тех, у кого нет заказов) с пустыми значениями для OrderID
2) Запрос вернёт только заказы, без клиентов
3) ❌ Запрос вернёт только клиентов с заказами
4) Запрос не вернёт результатов
  
### ИИ анализ
Вы выбрали неверный вариант.  `LEFT JOIN` гарантирует, что все строки из левой таблицы (`Customers`) будут в результате.  Для клиентов без заказов не найдётся соответствия в правой таблице (`Orders`), поэтому в колонке `OrderID` будет `NULL` (пустое значение).  Правильный ответ — первый, он точно описывает поведение `LEFT JOIN`.  Важно запомнить, что `LEFT JOIN` не фильтрует строки левой таблицы.  

# Вопрос 4. Тема: Сложные запросы
  
**4. Какой результат вернёт следующий SQL-запрос?**

```
SELECT Name FROM Customers
UNION
SELECT Name FROM Suppliers;
```
  
Варианты ответов:
1) Объединит результаты обеих таблиц, включая дубликаты
2) ❌ Объединит результаты, но добавит NULL вместо отсутствующих значений
3) Вернёт только уникальные имена из таблиц Customers и Suppliers
4) Вернёт только совпадающие значения из обеих таблиц
  
### ИИ анализ
Вы выбрали неверный вариант.  Ключевое свойство оператора `UNION` по умолчанию — удаление дубликатов из конечного набора результатов.  Он не добавляет `NULL` и не требует совпадения значений в обеих таблицах.  Правильный ответ — третий: запрос вернёт один список всех уникальных имён, найденных в обеих таблицах.  Для включения дубликатов нужно использовать `UNION ALL`.  

# Вопрос 5. Тема: Сложные запросы
  
**5. Какой результат вернёт следующий SQL-запрос?**

```
SELECT ProductName FROM Products
EXCEPT
SELECT ProductName FROM DiscontinuedProducts;
```
  
Варианты ответов:
1) Вернёт только устаревшие товары
2) Вернёт только товары, которые присутствуют в обеих таблицах
3) Вернёт список всех товаров, включая устаревшие
4) ✅ Вернёт товары, которые есть в Products, но отсутствуют в DiscontinuedProducts
  
### ИИ анализ
Оператор `EXCEPT` (или `MINUS` в некоторых СУБД) возвращает строки из первого набора данных, которых нет во втором.  В данном контексте это идеально описывает выборку актуальных товаров, исключая снятые с производства.  Вы правильно определили логику операции, что показывает понимание работы с множествами в SQL.  Это полезно для фильтрации данных.
