## Блок 2. Каркас HTML и семантика

### Содержание и вопросы

- Что такое базовая структура HTML-документа?
- Зачем нужны `head` и `body`?
- Какие семантические теги подходят для шапки, основного контента и подвала?
- Как правильно использовать заголовки `h1`-`h3`?

### Материал (лаконично)

- `<!doctype html>` это “режим HTML”. Без него браузер может странно интерпретировать разметку.
- `head` это “настройки страницы”:
  - кодировка, чтобы текст на русском не ломался
  - `viewport`, чтобы страница нормально выглядела на телефоне
  - `title`, чтобы было понятное имя вкладки
  - подключение `styles.css`
- `body` это все, что видит пользователь.
- Семантические теги это понятные названия блоков вместо безликих контейнеров. Они помогают:
  - быстрее ориентироваться в коде
  - лучше понимать структуру страницы
  - подготовить основу для доступности
- Минимальный набор для страницы-визитки: `header` (шапка), `main` (основное), `footer` (подвал).
- Заголовки:
  - `h1` один на страницу, это главный смысл страницы
  - `h2` для секций
  - `h3` для элементов внутри секции, например для карточек

### Условия

Сделай каркас `index.html` пошагово:

1. Создай базовую структуру документа.
2. В теге `html` установи язык документа.
3. В `head` добавь:
   - кодировку
   - настройку `viewport`
   - заголовок вкладки
   - подключение `styles.css`
4. В `body` создай 3 блока:
   - шапка
   - основная часть
   - подвал
5. В шапке добавь:
   - главный заголовок страницы
   - короткую строку-описание
6. В основной части создай минимум 3 секции.
7. В каждой секции добавь заголовок.

Проверка:

- Главный заголовок один.
- Секций в основной части не меньше трех.

### Псевдокод решения

```text
CREATE HTML_DOCUMENT
  SET DOCUMENT_LANGUAGE
  ADD META_CHARSET
  ADD META_VIEWPORT
  SET PAGE_TITLE
  LINK STYLESHEET_FILE

  BODY
    HEADER
      MAIN_TITLE
      SUBTITLE

    MAIN
      SECTION_A
        SECTION_TITLE
        PARAGRAPH

      SECTION_B
        SECTION_TITLE
        CONTENT_PLACEHOLDER

      SECTION_C
        SECTION_TITLE
        CONTENT_PLACEHOLDER

    FOOTER
      CONTACTS_PLACEHOLDER
END
```
