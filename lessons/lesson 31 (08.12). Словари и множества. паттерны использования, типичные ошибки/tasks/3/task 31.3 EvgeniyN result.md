### Отчёт по проверке решения задания

**1. Формулировка задания**  
Требуется определить общие темы воркшопов для новичков (`set_a`) и наставников (`set_b`), а также темы, уникальные для наставников.  
- **Входные данные**: два множества строк (названия тем).  
- **Ожидаемый вывод**:  
  - Множество общих тем с подписью `Совпадения:`.  
  - Множество тем только для наставников с подписью `Только наставникам:`.  
- **Ключевые правила**:  
  - Использовать операции над множествами (`&`, `-`, `|`).  
  - Не преобразовывать множества в списки без необходимости.  
  - Выводить осмысленные подписи.

---

**2. Результаты проверки**  
**Файл**: `PR_31_3.py`  
**Код студента**:
```python
set_a = {'python', 'git', 'design'}
set_b = {'mentoring', 'python', 'pitch'}
print(f"Совпадение:", set_a & set_b)  
print(f"Толко наставникам:", set_b - set_a) 
```

**Тестирование**:  
- Запуск кода с примером из условия выводит:
  ```
  Совпадение: {'python'}
  Толко наставникам: {'mentoring', 'pitch'}
  ```
- **Ошибки воспроизводятся** при любых входных данных из-за опечаток и некорректного форматирования.

**4. Ошибки**  

**Значимые** (влияют на соответствие условию):  
1. **Некорректные подписи в выводе**:  
   - Опечатка: `"Толко наставникам:"` → должно быть `"Только наставникам:"`.  
   - Несоответствие примеру: `"Совпадение:"` (единственное число) → должно быть `"Совпадения:"` (множественное число).  
   **Исправление**:  
   ```python
   print(f"Совпадения: {set_a & set_b}")  
   print(f"Только наставникам: {set_b - set_a}")
   ```

**Минорные** (стилевые недочёты):  
1. **Некорректное использование f-строк**:  
   В коде `f"Совпадение:", set_a & set_b` значение множества передаётся как отдельный аргумент `print`, а не внутри f-строки.  
   **Исправление**:  
   ```python
   print(f"Совпадения: {set_a & set_b}")
   ```
