# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Комьюнити-кухня отслеживает, кто последний пополнил полку ингредиентов. Координатор быстро проверяет, кто отвечает за конкретную полку и нужно ли напоминание. Создай словарь restock_by, где ключ — название полки, а значение — имя последнего волонтёра. Затем выведи информацию для переменных check_shelf и empty_shelf.

## Пример входных данных
```python
shelves = [
    ('cereals', 'Аня'),
    ('spices', 'Илья'),
    ('fruits', 'Маша'),
]
check_shelf = 'spices'
empty_shelf = 'tea'
```

## Пример выходных данных
```text
spices -> Илья
tea -> пока никто
```

## Критерии успешного выполнения
- Список пар преобразован в словарь restock_by с ключом 'название полки'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для check_shelf и empty_shelf в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку shelves в цикле и наполняй словарь restock_by.
- Используй множественное присваивание: key, value = запись из shelves.
- Для отсутствующих данных возвращай строку 'пока никто'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
