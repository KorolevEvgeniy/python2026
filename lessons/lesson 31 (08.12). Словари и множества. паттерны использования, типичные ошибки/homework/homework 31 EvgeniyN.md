## Задача 1. Волонтёры и полки

- **Блок:** словарь как структура быстрого доступа, `get`.

В пункте выдачи вещей волонтёры закреплены за полками.

Пример входных данных:
```python
shelves = [
    ("toys", "Игорь"),
    ("books", "Анна"),
    ("clothes", "Мария"),
]
check_shelf = "books"
empty_shelf = "electronics"
```

### Требуется
1. Собрать словарь `restock_by`, где ключ. название полки, значение. имя волонтёра.
2. Вывести две строки:
   ```text
   <check_shelf> -> <волонтёр или "пока никто">
   <empty_shelf> -> <волонтёр или "пока никто">
   ```
3. Для чтения из словаря использовать только `get(name, "пока никто")`.

### Критерии
- Формат строк полностью соответствует примеру.
- Нет вывода `None` вместо осмысленного текста.

## Задача 2. Учёт поездок по станциям

На уроке у тебя были небольшие ошибки в текстах сообщений и лишний вывод. Здесь важно привести формат к требуемому.

Пример входных данных:
```python
logs = [
    ("park", 12),
    ("river", 7),
    ("park", 5),
    ("mall", 3),
]
finished = ["mall", "unknown_station"]
```

### Требуется
1. Собрать словарь `rides`, где ключ. название станции, значение. суммарное количество поездок.
2. Для каждой станции из `finished` удалить запись из словаря с помощью `pop(name, 0)`.
3. Ничего лишнего не печатать во время удаления.
4. После цикла вывести только итоговый словарь `rides`.

### Критерии
- Суммирование значений реализовано через `get`.
- `pop` вызывается с запасным значением и не вызывает ошибок.
- В выводе нет промежуточных отладочных строк.

## Задача 4. Подготовка к comprehensions

Нужно собрать статистику по отчётам, чтобы в следующем уроке можно было переписать решение через генераторы коллекций.

Пример входных данных:
```python
reports = [
    ("daily", 5),
    ("weekly", 3),
    ("daily", 2),
    ("monthly", 1),
]
limit = 3
```

### Требуется
1. Собрать словарь `report_counts`, где ключ. тип отчёта, значение. суммарное число отчётов.
2. Построить словарь `popular`, куда попадут только типы с количеством не меньше `limit`.
3. Распечатать оба словаря с понятными подписями.

### Критерии
- Решение разбито на два понятных шага.
- Формат вывода чёткий, без «магических» чисел в тексте.
- Логику легко переписать в comprehension.
