# Вопрос 17. Тема 3: Рекурсия

Варианты ответов:
1) Станет быстрее
2) Выполнится один раз
3) Python сам остановит
4) ✅ Будет ошибка RecursionError

### ИИ анализ
При отсутствии условия остановки рекурсивная функция будет вызывать саму себя бесконечно.  В Python это приводит к переполнению стека вызовов и возникновению исключения `RecursionError`, которое прерывает выполнение программы.  Это одна из самых частых ошибок при работе с рекурсией.

# Вопрос 36. Тема 6: Основы асинхронного программирования

Что обычно означает «асинхронность» в программировании?

Варианты ответов:
1) ✅ Не ждать одну задачу
2) Только про файлы
3) Всегда быстрее
4) Только одно ядро
5) Нельзя тестировать

### ИИ анализ
Асинхронность означает, что программа может приостановить выполнение одной задачи, если она ожидает ввода-вывода (например, ответа от сети), и переключиться на другую задачу. Это повышает общую эффективность, позволяя не блокировать поток выполнения в ожидании. Она не гарантирует ускорения вычислений и не ограничивается работой с файлами.

Она позволяет эффективно управлять множеством таких операций, не блокируя выполнение программы. Для CPU-интенсивных задач (тяжелая математика) она не даёт преимуществ.

# Вопрос 39. Тема 6: Основы асинхронного программирования

Варианты ответов:
1) Завершает программу
2) Создает список
3) Импортирует модуль
4) ✅ Ожидает завершения асинхронной операции

### ИИ анализ
Ключевое слово `await` приостанавливает выполнение текущей асинхронной функции (корутины) до тех пор, пока переданный ему awaitable-объект (например, другая корутина) не завершит свою работу. Это позволяет освободить цикл событий для выполнения других задач, пока текущая ожидает результата. `await` можно использовать только внутри функций, объявленных как `async def`.

# Вопрос 40. Тема 6: Основы асинхронного программирования
