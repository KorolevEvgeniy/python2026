# Задание 33.2. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«List comprehensions. базовый рабочий инструмент» — гибкое построение списков из нескольких источников с фильтрацией и переупорядочиванием данных.

## Условие задачи
**Сложность**: <span style="color: #c62828">● сложный</span>. примерное время: 15–20 минут

Тебе нужно объединить данные из двух списков: `films` (название, год) и `ratings` (название, балл). Сформируй один list comprehension, который создаёт список строк `"Название (год) — балл"` только для фильмов после 2015 года с рейтингом ≥ 7.5. Считай, что названия совпадают и встречаются не более одного раза в каждом списке. Используй вложенный источник данных и фильтр в comprehension, избегая дополнительных структур.

## Пример входных данных
```
films = [
    ("Нолан", 2014),
    ("Прибытие", 2016),
    ("Дюна", 2021)
]
ratings = [
    ("Нолан", 6.9),
    ("Прибытие", 8.0),
    ("Дюна", 7.4)
]
```

## Пример выходных данных
```
['Прибытие (2016) — 8.0', 'Дюна (2021) — 7.4']
```

## Критерии успешного выполнения
- Вся логика реализована одним comprehension с несколькими источниками.
- Отбор по году и рейтингу работает одновременно.
- Формат строки соответствует образцу.
- Нет промежуточных словарей или циклов.

## Подсказки
- Внутри comprehension можно перебирать пары из `films`, а затем находить соответствующий рейтинг через дополнительный генератор и `next`.
- Либо используй двойной `for` и условие совпадения названий, но следи за читаемостью.

## Частые ошибки и ограничения
- Дублирование фильмов из-за отсутствия проверки названия.
- Создание внешних структур нарушает требование одной конструкции.
- Не модифицируй исходные списки.
