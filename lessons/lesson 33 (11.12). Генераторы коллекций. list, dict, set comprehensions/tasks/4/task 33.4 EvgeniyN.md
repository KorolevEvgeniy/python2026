# Задание 33.4. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Set comprehensions» дают возможность собирать уникальные значения и фильтровать элементы в одной компактной конструкции.

## Условие задачи
**Сложность**: <span style="color: #c62828">● сложный</span>. примерное время: 15–20 минут

Есть список словарей `tracks`, где каждый словарь содержит ключи `"title"`, `"tags"`, `"released"`. Поле `"tags"` — это список строк. Построй одним set comprehension множество `focus_tags`, в которое попадут теги, встречающиеся хотя бы у двух треков, выпущенных после 2018 года. Используй только comprehension и базовые операции над множествами.

## Пример входных данных
```
tracks = [
    {"title": "Airwaves", "tags": ["synth", "ambient"], "released": 2019},
    {"title": "Dust", "tags": ["ambient", "cinematic"], "released": 2021},
    {"title": "Retro", "tags": ["synth", "pop"], "released": 2016}
]
```

## Пример выходных данных
```
{'ambient'}
```

## Критерии успешного выполнения
- Множество `focus_tags` формируется одним set comprehension.
- Учитываются только треки с `released > 2018`.
- В результате остаются теги, которые встретились минимум два раза среди отобранных треков.
- Исходный список `tracks` не изменяется.

## Подсказки
- Можно использовать вспомогательный словарь с подсчётами перед comprehension, но финальная структура должна строиться comprehension-ом.
- Для проверки частоты воспользуйся структурой `count[tag] >= 2`.
- Следи, чтобы теги не дублировались: множество уже устраняет повторы.

## Частые ошибки и ограничения
- Добавление `set()` вокруг comprehension не требуется.
- Не включай треки из старых годов.
- Не допускай, чтобы списки содержали больше пяти пунктов.
