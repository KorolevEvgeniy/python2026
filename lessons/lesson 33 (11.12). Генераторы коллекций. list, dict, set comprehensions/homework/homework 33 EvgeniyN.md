## Задача 1. Повторение сложной темы — «активные станции метро»
- **Блок:** list comprehension + форматирование.
- **Описание:** дан список `stations = [{"name": "Тимирязевская", "load": [45, 60, 80]}, ...]`. Нужно создать список строк `"Станция — период (значение%)"`, где:
  - в `load` три числа: утро/день/вечер;
  - в результат попадают станции, где впервые превышено 70%;
  - `periods = ["утро", "день", "вечер"]`;
  - формат вывода: `"<имя> — <период> (<значение>%)"`.
- **Критерии:** один list comprehension, поиск первого превышения реализован в выражении, формат строк соблюдён (длинное тире и пробелы).

## Задача 2. Закрепление практики — «треки выше среднего»
- **Блок:** list comprehension с пересчётом среднего.
- **Описание:** есть два списка:
  ```python
  tracks = [("UNNQ", "Pulse"), ("Horizon", "Flight")]
  plays = [("Pulse", 95), ("Flight", 110)]
  ```
  - Вычисли среднее количество прослушиваний один раз.
  - Построй список строк `"Исполнитель — Название (N прослушиваний)"` для треков с `N > average`.
  - Учти, что названия в `tracks` и `plays` совпадают.
- **Критерии:** среднее считается вне comprehension, формат строки строго как в примере (пробел перед «прослушиваний»), один comprehension без вложенных `append`.

## Задача 3. Интеграция блоков — «нагрузка хабов»
- **Блок:** dict comprehension + суммирование.
- **Описание:** список `hubs = [{"name": "DevHub", "capacity": 200, "streams": [{"viewers": 120}, ...]}, ...]`.
  - Собери словарь `hub_load`, ключ — название, значение — коэффициент загрузки `round(sum(viewers)/capacity, 1)`.
  - Включай только хабы с ≥2 потоками.
- **Критерии:** один dict comprehension, используется сумма зрителей (не среднее), округление до одного знака, исходный список не изменяется.

## Задача 4. Подготовка к следующему уроку — «расписание ревью»
- **Блок:** комбинация dict и set comprehension.
- **Описание:** дан список строк `events = ["backend:code-review:15", "android:deploy:45", ...]`.
  1. Построй множество `rapid_departments` из департаментов, где действие `code-review` или `hotfix` и длительность < 20.
  2. Построй словарь `review_slots`, где ключ — департамент, значение — список длительностей его ревью, но только если все ревью ≤ 60 минут.
- **Требования:** строки всегда имеют формат `dept:type:minutes`. Игнорируй записи с другим форматом.
- **Критерии:** каждый шаг использует comprehension, длительность преобразуется в `int` внутри выражения, нет побочных эффектов.
