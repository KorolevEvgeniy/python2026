# Вопрос 1. Тема: Форматы JSON и CSV
  
**1. Что из перечисленного НЕ является допустимым типом данных в JSON?**
  
Варианты ответов:
1) Строка
2) Словарь
3) ✅ Функция
  
### ИИ анализ
Правильный ответ — «Функция». JSON (JavaScript Object Notation) — это текстовый формат для обмена данными, который поддерживает простые типы: строки, числа, логические значения, null, массивы и объекты (словари). Функции являются исполняемым кодом и не могут быть сериализованы в JSON, что делает их недопустимым типом данных в этом формате. Важно помнить, что JSON предназначен для данных, а не для кода.

# Вопрос 2. Тема: Форматы JSON и CSV
  
**2. Какой параметр позволяет улучшить читаемость JSON-файлов при записи?**
  
Варианты ответов:
1) newline
2) ✅ indent
3) formatter
  
### ИИ анализ
Правильный ответ — «indent». При использовании функции `json.dump()` или `json.dumps()` параметр `indent` определяет количество пробелов для отступов, что форматирует вывод JSON с вложенными структурами, делая его удобочитаемым для человека. Параметр `newline` относится к управлению концами строк в файлах, а `formatter` — это нестандартный параметр для сериализации JSON в Python.

# Вопрос 3. Тема: Форматы JSON и CSV
  
**3. Какой метод используется для записи данных в JSON-файл?**
  
Варианты ответов:
1) ✅ json.dump()
2) json.save()
3) json.write()
  
### ИИ анализ
Правильный ответ — `json.dump()`. Этот метод принимает объект Python и файловый объект, затем записывает JSON-представление объекта в этот файл. Методы `json.save()` и `json.write()` не существуют в стандартной библиотеке Python для работы с JSON. Для преобразования объекта в JSON-строку (без записи в файл) используется `json.dumps()`.

# Вопрос 4. Тема: Форматы JSON и CSV
  
**4. Какой метод используется для записи данных в CSV-файл?**
  
Варианты ответов:
1) ❌ csv.dump()
2) csv.read()
3) csv.writerows()
  
### ИИ анализ
Выбранный вариант `csv.dump()` неверен. Правильный ответ — `csv.writerows()`. Для записи данных в CSV в Python используется объект `writer`, созданный функцией `csv.writer()`. Метод `writerows()` этого объекта принимает список строк (или итерируемый объект) и записывает их все в файл. Метод `csv.read()` используется для чтения, а `csv.dump()` не существует в модуле `csv`.

# Вопрос 5. Тема: Форматы JSON и CSV
  
**5. Какой из следующих форматов поддерживает вложенные структуры данных?**
  
Варианты ответов:
1) ✅ JSON
2) CSV
3) Оба
  
### ИИ анализ
Правильный ответ — «JSON». Формат JSON изначально поддерживает вложенные объекты и массивы, что позволяет представлять сложные иерархические данные. CSV (Comma-Separated Values) — это плоский табличный формат; каждая строка представляет собой запись, а столбцы — поля, поэтому напрямую вложенные структуры в нем не поддерживаются. Для хранения сложных данных в CSV их обычно нужно «расплющить».

# Вопрос 6. Тема: Форматы JSON и CSV
  
**6. Выберите верные утверждения о JSON (выбор нескольких вариантов):**
  
Варианты ответов:
1) ❌ JSON поддерживает вложенные структуры
2) JSON поддерживает числовые типы данных
3) JSON можно использовать только в Python
4) JSON не поддерживает тип данных null
  
### ИИ анализ
В данном ответе отмечено только одно утверждение как верное, и оно неверно. Давайте разберем все варианты: 1) **Верно**, JSON поддерживает вложенные структуры через объекты и массивы. 2) **Верно**, JSON поддерживает числовые типы (целые и с плавающей точкой). 3) **Неверно**, JSON — это независимый от языка формат, используемый во многих технологиях. 4) **Неверно**, JSON поддерживает тип `null`. Таким образом, правильными являются утверждения 1 и 2.
