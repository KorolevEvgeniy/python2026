# Практическое задание 35.1

## Задания

### Описание задачи
На предыдущих уроках вы изучили коллекции и алгоритмы. Теперь мы переходим к основам функций. Этот блок посвящен базовому пониманию объявления функций, передачи аргументов и использования оператора `return`. Ваша задача — научиться отличать печать от возврата значения, что является фундаментом для работы с любыми функциями.

Вы — разработчик, помогающий автоматизировать расчеты для небольшого магазина. Вам нужно создать функцию, которая вычисляет общую стоимость покупки. Пока что в магазине продается только один вид товара, но в разном количестве.

Создайте функцию `calculate_total`, которая принимает два аргумента:
1. `price` — цена одной единицы товара (число).
2. `quantity` — количество товара (целое число).

Функция должна **возвращать** результат умножения цены на количество. Не используйте `print` внутри функции для вывода результата — функция должна возвращать значение, чтобы его можно было использовать дальше в программе (например, для печати или других расчетов).

### Пример входных и выходных данных
**Пример 1:**
*   **Входные данные для вызова функции:** `price=100`, `quantity=5`
*   **Что происходит:** Функция получает числа 100 и 5.
*   **Ожидаемое возвращаемое значение функции:** `500`
*   **Как это может выглядеть в коде:**
    ```python
    total_cost = calculate_total(100, 5)
    print(total_cost)  # Эта команда напечатает: 500
    ```

**Пример 2:**
*   **Входные данные для вызова функции:** `price=75.5`, `quantity=2`
*   **Ожидаемое возвращаемое значение функции:** `151.0`
*   **Как это может выглядеть в коде:**
    ```python
    result = calculate_total(75.5, 2)
    print("Итог:", result)  # Эта команда напечатает: Итог: 151.0
    ```

### Критерии проверки и ограничения
*   Функция должна быть объявлена с помощью `def`.
*   Функция должна иметь два параметра: `price` и `quantity`.
*   Внутри функции должен использоваться оператор `return` для возврата результата вычисления.
*   Функция не должна содержать вызовов `print` для вывода итоговой суммы.
*   Аргументы при вызове функции передаются позиционно (по порядку).
*   Не используйте темы из будущих блоков: значения по умолчанию, именованные аргументы, `*args`, `**kwargs`, `global`.

### Решение задачи (псевдокод)
1.  Объяви функцию с именем `calculate_total`, которая принимает два параметра.
2.  Внутри функции вычисли произведение первого параметра на второй.
3.  Верни результат этого вычисления из функции с помощью соответствующего оператора.
4.  За пределами функции вызови ее, передав два числовых аргумента.
5.  Сохрани результат вызова в переменную.
6.  Напечатай значение этой переменной, чтобы увидеть результат работы функции.

### Рекомендации
*   **Вынести в отдельную функцию:** В данном случае вся логика задачи уже инкапсулирована в одну функцию `calculate_total`. Дополнительного вынесения не требуется.
*   **PEP8:**
    *   Используйте осмысленные имена для функции и параметров (как в задании — `calculate_total`, `price`, `quantity`).
    *   После объявления функции (`def`) и перед кодом внутри нее оставляйте две пустые строки. После окончания тела функции также оставляйте две пустые строки перед следующим блоком кода.
